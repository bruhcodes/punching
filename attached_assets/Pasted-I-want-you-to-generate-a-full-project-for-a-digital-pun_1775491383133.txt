### **USER FLOW**

1. User opens app → sees install prompt
2. User continues → onboarding form:
   - Input:
     - Name
     - Phone number

3. On submit:
   - Create user in Supabase:
     - id (UUID)
     - name
     - phone
     - punch_count = 0
     - total_punches = 10
     - created_at timestamp
   - Generate a QR code for that user (used for admin lookup)

4. User dashboard:
   - Show a **digital punch card**
   - Display:
     - Punch progress (stamps, stars, or progress bar)
     - QR code (for admin to scan)
   - Clean Apple-style UI
   - Show basic profile info

### **ADMIN DASHBOARD (FULL CONTROL)**

Admin must be able to access EVERYTHING.

1. Dashboard:
   - View ALL users

     - Name
     - Phone
     - ID
     - OR scan QR code

2. User Details Page:
   - Show:

     - Total punches

   - Buttons:
     - “Add Punch”

     - “Reset Punches”

   - When admin clicks “Add Punch”:
     - Increase punch_count

5. Notifications Tab:

### **DATABASE (SUPABASE)**

**Users Table**
- id (UUID)
- name (text)
- phone (text)
- punch_count (int)
- total_punches (int)
- qr_code (text)
- created_at (timestamp)

**Notifications Table**
- id (UUID)
- user_id (UUID)
- message (text)
- created_at (timestamp)
- read (boolean)

**Settings Table (for styles)**
- id
- stamp_type
- background_style
- progress_style

---

### **APIs**

- POST /users → create user
- GET /users → get all users (admin)
- GET /users/:id → get single user
- POST /users/:id/punch → add punch
- POST /users/:id/remove → remove punch
- POST /users/:id/reset → reset punches
- POST /notifications → send notification
